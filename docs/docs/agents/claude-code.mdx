# Claude Code

[Claude Code](https://code.claude.com) is Anthropic's agentic coding CLI. HackAgent treats a **locally installed** Claude Code as a first-class attack target — it drives it natively through the `claude-code` router provider, which shells out to the headless `claude -p` CLI.

There is **no HTTP endpoint or bridge** to stand up: the only prerequisite for the target is the `claude` binary on your `PATH`. HackAgent registers a per-instance LiteLLM `CustomLLM` whose `completion` runs `claude -p --output-format json` instead of making an HTTP call, so the exchange still flows through the standard tracking pipeline like every other provider.

## Prerequisites

1. **Install Claude Code** and confirm it runs:
   ```bash
   claude --version
   ```
2. **(Optional) An Anthropic key for the attacker/judge.** The *target* (Claude Code) authenticates itself through its own CLI — it needs no key from HackAgent. Only the attacker and judge models need one, and only if you route them to the Anthropic API:
   ```bash
   export ANTHROPIC_API_KEY="sk-ant-..."
   ```
   If you'd rather not use a key at all, point the attacker/judge at a local Ollama model instead (see [Fully local runs](#fully-local-runs)).

## Quick Start

The fastest way is the bundled `hackagent claude` preset, which pre-fills the target, model, attack strategy (FlipAttack), and a starter set of red-team goals. FlipAttack is the default because it uses no external attacker model — each goal is a single obfuscated turn against the target, so a run is much faster than a search-based attack like TAP.

Opens the interactive TUI with the Attacks tab pre-configured to target Claude Code. Edit the goals and hit execute.

## Fully local runs

`hackagent claude` only specifies the *target*. By default the attacker and judge fall back to HackAgent's local Ollama configuration, so if you have Ollama running you can red-team Claude Code **end to end without any Anthropic key**:

```bash
ollama serve   # in another shell; the default abliterated model is pulled automatically
hackagent claude --no-tui
```

To route the attacker/judge to the Anthropic API instead (as the example script does), set `ANTHROPIC_API_KEY` and pass them explicitly in `attack_config`:

```python
attack_config = {
    "attack_type": "tap",
    "goals": [...],
    "attacker": {"identifier": "anthropic/claude-sonnet-4-6", "agent_type": "litellm", "endpoint": ""},
    "judge":    {"identifier": "anthropic/claude-sonnet-4-6", "agent_type": "litellm", "endpoint": "", "type": "harmbench"},
}
```

## Configuration

The target is configured through `adapter_operational_config`:

| Key | Default | Description |
|-----|---------|-------------|
| `name` | *(required)* | Claude model for `claude --model` (`sonnet`/`opus`/`haiku` alias or a full id like `claude-opus-4-8`). |
| `binary` | `claude` | Path to the Claude Code executable. |
| `system_prompt` | – | Override the system prompt (`--system-prompt`). |
| `append_system_prompt` | – | Extend the system prompt (`--append-system-prompt`). |
| `max_turns` | – | Cap the agentic loop iterations (`--max-turns`). |
| `cwd` | – | Working directory to run `claude` in. |
| `timeout` | `300` | Per-turn timeout in seconds. |
| `extra_args` | `[]` | Additional raw `claude` flags. |

:::note Prompt safety
The adversarial prompt is fed to `claude -p` over **stdin** (never argv), so text that begins with `-` is never misread as a CLI flag, and long prompts sidestep argv length limits.
:::

:::note Tools & permissions
The target runs with Claude Code's default permission mode. In headless mode, permission-gated tools (bash, file writes) do not execute, so this exercises the model's safety behaviour without granting it actions on your machine. Pass extra CLI flags via `extra_args` if you intentionally want a different posture.
:::

## Troubleshooting

### `claude` not found on PATH

```
ClaudeCodeConfigurationError: Claude Code executable 'claude' was not found on PATH.
```

The provider verifies the binary at construction, so a missing install fails fast. Install Claude Code from [code.claude.com](https://code.claude.com), or pass the full path via `--binary` / `adapter_operational_config["binary"]`.

### Attacker/judge errors about a missing API key

This means the attacker/judge are routed to the Anthropic API but `ANTHROPIC_API_KEY` isn't set. Either export the key, or use a local Ollama attacker/judge — see [Fully local runs](#fully-local-runs). Remember the *target* never needs a key.
